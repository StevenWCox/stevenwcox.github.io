Logger

global inherited sharing class Logger
DescriptionThe Logger's primary responsibility is to create Log records. It utilizes immediate Platform Events to circumvent rollbacks that happen when an exception is thrown in a transaction. After calling Logger.log, you can be assured that the log record will be committed. An additional bonus that you get for free: a batchable class that implements Database.RaisesPlatformEvents will auto-generate a log record when an exception is thrown.

An additional benefit of Logger is that it can be turned on and off via hierarchical Apex Boost custom settings. Just adjust the Log Level for any user or profile - even in production - to start creating log records in order to triage problems.

In addition to the object you pass to Logger.log, Apex Boost will record the context (class, method, line), current user, and stack trace.

Finally, once you've implemented logging, stakeholders will be able to build reports and workflow to visualize and be notified of errors occurring in the system.

Usage:

 // ERROR|scox~ MyClass.myMethod:14 simple test
 aBoost.Logger.log('simple test');
 

 // WARN|scox~ MyClass.myMethod:16 warning test
 aBoost.Logger.log(LoggingLevel.WARN, 'warning test');
 

 // ERROR|Class.MyClass.execute:16
 Database.executeBatch(new MyClass());

Logger.Cleaner

global without sharing class Cleaner extends BatchJob.Handler
DescriptionA batch process that deletes old log records. On initial installation of Apex Boost, this process is scheduled to run at 1am every day. The Log Days To Keep custom setting field specifies how many days log records will be kept. The default is 30.

Any exceptions thrown by Logger.Cleaner will be logged by Logger.

Usage:

 // schedule the job to run every day at 1am
 System.schedule('Clean Log Records', '0 0 1 * * ?', new aBoost.BatchJob(new aBoost.Logger.Cleaner()));
 

 // clean old logs immediately
 new aBoost.Logger.Cleaner().run();
