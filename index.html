Home

Welcome to the ApexBoost Library - a collection of time-saving, high-quality code for use in any Salesforce project. If you've spent any time writing C++, you may be familiar with the Boost libraries. Boost.org's answer to "Why should an organization use Boost?" is
In a word, Productivity. Use of high-quality libraries like Boost speeds initial development, results in fewer bugs, reduces reinvention-of-the-wheel, and cuts long-term maintenance costs. And since Boost libraries tend to become de facto or de jure standards, many programmers are already familiar with them.
ApexBoost has been created with similar goals and capabilities in mind. It contains patterns, tools, and utilities that have been used in dozens of successful projects over many years. Below is a summary of the major features, sorted roughly by general usefulness and impact. Additions will be released often!

Logging

The lws.Logger generates log records when error or debug events occur in custom code. Logs will be saved even if a subsequent exception is thrown - guaranteeing that errors will be noticed. Stakeholders can schedule reports or create other workflows to receive notifications in real time.

Debugging

A suite of debugging utilities provides auto-generated context, boosting developer productivity and slashing debug time.

Testing

Unit testing helpers and utilities in lws.Test save time and encourage consistency and best practices.

Assertions

Assertions within code (not just unit tests) are a tremendous way to assure quality and find bugs quickly. ApexBoost lws.Assert provides catchable assertion methods that can be disabled without altering the code.

Trigger Handling

lws.TriggerHandler provides a simple, intuitive, and powerful trigger framework to make adding handlers a breeze. Triggers can be disabled individually or in bulk - for a particular user, profile, or the entire org.

Queries

SOQL statements are a frequent source of errors due to typos or ignoring CRUD or FLS security concerns. lws.Query encapsulates a query, preventing typos and doing all the security checks in the background.

Batch Jobs

lws.BatchJob introduces a very convenient pattern for declaring your batchable and schedulable business logic. Rather than creating multiple Apex classes, simply add a nested class that extends lws.BatchJob.Handler. The rest happens auto-magically!

The ApexBoost Library is built using our high internal standards and Salesforce best practices. With each release, we guarantee:

  • A full suite of lightning-fast unit tests with 100% coverage.
  • Clean PMD and CPD reports, guarantying security, scalability, and maintainability.
  • A clean Checkmarx security scan.
  • Internal diagnostics and error reporting using the library itself.
