Logger

global inherited sharing class Logger
Description

The lws.Logger's primary responsibility is to generate a log event, which will be saved as a lws__Log record. The log level can be adjusted in the "Settings" custom settings.

Usage:

 // ERROR|scox~ MyClass.myMethod 1 simplest test
 lws.Logger.log('simplest test');
 

 // WARN|scox~ MyClass.myMethod 1 warning test
 lws.Logger.log(LoggingLevel.WARN, 'warning test');
 

 // ERROR|Class.MyClass.execute: line 16, column 1
 Database.executeBatch(new MyClass());

Logger.Cleaner

global without sharing class Cleaner extends BatchJob.Handler
Description

A batch process that deletes old log records. The Util "Log Days To Keep" custom setting specifies how many days of logs will be kept. The default is 30.

Any batch errors will be logged.

Usage:

 // schedule the job to run every day at 1am
 System.schedule('Clean LWS Log Records', '0 0 1 * * ?', new lws.BatchJob(new lws.Logger.Cleaner()));
 

 // clean old logs immediately
 new lws.Logger.Cleaner().run();

BatchJob

global inherited sharing class BatchJob implements Database.Batchable<sObject>, Database.AllowsCallouts, Database.RaisesPlatformEvents, Database.Stateful, Schedulable
Description

A generic class for scheduled batch operations. The goal here is to be able to reduce Apex code by providing a general utility that handles all the plumbing (debug logging, error logging, etc) and delegates the interesting work to domain sub-classes. To do so, create a class that extends lws.BatchJob.Handler and override start and execute.

 public class Accounts {
   public class Updater extends lws.BatchJob.Handler {
     public Database.QueryLocator start(Database.BatchableContext context) {
       return new lws.Query(Account.SObjectType);
     }
     public void execute(Database.BatchableContext context, SObject[] scope) {
       new lws.SObjectDomain(scope).dmlUpdate();
     }
   }
 }

Usage:

 // execute a batch update job
 new Accounts.Updater().run();
 

 // execute a batch with a custom batch size
 Database.executeBatch(new lws.BatchJob(new Accounts.Updater()), 5);
 

 // schedule the job with a custom batch size, starting in 10 minutes
 System.scheduleBatch(new lws.BatchJob(new Accounts.Updater(), 5), 'updating...', 10);
 

 // schedule the job with a custom batch size; run every day at 1pm
 System.schedule('updating...', '0 0 13 * * ?', new lws.BatchJob(new Accounts.Updater(), 5));

In addition to Batchable and Schedulable, this class implements Database.AllowsCallouts, Database.RaisesPlatformEvents, and Database.Stateful.

Query

global inherited sharing class Query
DescriptionUsing this class, a dynamic query can be constructed piece by piece, adding object and field type checking, which avoids typos and references to invalid fields. Object read permission and FLS may also optionally be checked.

Usage:

 // get user names where City starts with "San"
 User[] users = new lws.Query(User.SObjectType)
                       .field(User.Name)
                       .whereOp(User.City, 'LIKE', 'San%')
                       .run();
 

 // get the number of contacts in Dallas accounts
 Integer n = new lws.Query(Contact.SObjectType)
                    .field(Contact.Name)
                    .whereIn(Contact.AccountId,
                       new lws.Query(Account.SObjectType)
                              .whereOp(Account.City, '=', 'Dallas'))
                    .count();
 

 // get the contacts in certain accounts
 Account[] results = new lws.Query(Account.SObjectType)
                            .fields(Query(Account.Contacts, true))
                            .whereIn(Account.Id, accounts)
                            .run();
